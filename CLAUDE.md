# CLAUDE.md

## 패키지 매니저

항상 `pnpm`을 사용한다. `npm`이나 `yarn` 사용 금지.

## 개발 스크립트 규칙

새 기능을 구현할 때마다 아래 기준으로 개발 스크립트가 필요한지 판단한다.

**스크립트를 만들어야 하는 경우:**
- 새 API 엔드포인트가 생겼는데, 수동으로 테스트하려면 인증 토큰 + 특정 데이터가 필요한 경우
- 특정 기능을 확인하려면 DB에 시드 데이터가 필요한 경우
- 반복적으로 실행할 법한 curl/psql 작업이 생긴 경우

**스크립트를 만들 필요 없는 경우:**
- 기존 스크립트(`dev-reset-all.sh`, `dev-create-project.sh` 등)로 충분히 커버되는 경우
- 단순 리팩토링이나 버그 픽스

**스크립트를 추가할 때 따르는 규칙:**
1. `scripts/` 디렉토리에 `dev-<기능명>.sh` 형식으로 생성
2. 스크립트 상단에 `set -euo pipefail` 추가
3. `scripts/README.md`에 "어떤 상황에 쓰는 스크립트인지" 설명 추가
4. `package.json`의 `scripts`에 `pnpm` 단축 명령어 추가
